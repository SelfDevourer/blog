基本构成

  • 一个程序包括一个或多个函数，但只能有一个主函数——名字是main；
  • 主函数可以调用任何其他函数，任何其他函数可以互相调用但不能调用主函数；

基本数据类型

  • 构造类型是指由若干个相关的基本类型数据组合在一起形成的一种复杂的数据类型；
  • 数组是由同类型数据组合而成；
  • 结构型可以由不同数据类型组合而成；
  • 指针型用于表示内存地址；
  • 空类型表示不需要数据值，因此没有类型；

常量

常量、常数、字面常量是同一个意思，指程序运行过程中其值不改变的数据，常量有整形常量、实行常量、字符常量、字符串常量；

  • 整形常量可以存入十进制整数、八进制整数（以 0 结尾）和十六进制（以 0x 结尾）
  • 实型常量只使用十进制，有两种形式：
    • 一般形式：由整数、小数点和小数组成；
    • 指数形式：由指数、大小字母E、尾数组成，指数必须十进制整形。形式如”尾数E/e指数“，计算公式为：尾数 X 10的指数次方，例如1.1E2，0e0，1E1。详细解释：科学记数法和指数运算；
  • 字符常量是两个单引号'括住的单个字符；
  • 字符串常量是两个双引号“括住的一串字符；
  • 符号常量是常量的另一种表示方法，定义规则： #define 符号常量 常量。符号常量的定义规则属于“宏定义命令“，在程序中使用定义过的“宏名“称为“宏调用“。在编译前会将符号常量替换成对应的常量，称为“宏替换“，宏定义命令属于“预编译命令“；

变量

  • 字符串只能是常量；
  • 被定义为整数的变量的值如果在-128～127之间可以当作字符型变量使用；
  • 被定义为字符型的变量可以当作整形使用，也可以被当作无符号（unsigned）整形使用，值在0～255之间；
  • 定义变量后会自动分配与其类型相配数量的连续内存单元；
  • 有名常量的定义规则： const 数据类型符 变量名 = 初值;；

运算符

  • 运算符有一个运算对象称为单目运算符，放在运算对象前面称为前缀单目运算符，放在后面称为后缀单目运算符；
  • 运算符有两个运算对象称为双目运算符，又称双目中缀运算符；
  • 运算符有三个运算对象称为三目运算符；
  • 每个运算符运算的对象和运算结果有确定的数据类型；
  • 运算符有分为基本运算符和专用运算符，
    • 基本运算符有：
      • 算术运算符：+、-、*、／、%、++、--
      • 关系运算符：<、<=、>、>=、==、!=
      • 逻辑运算符！、&&、||
      • 赋值运算符：=、+=、-+、*=、／=、%=
      • 逗号运算符：，
      • 数据长度运算符：sizeof()
      • 位运算符：～、&、|、^、>>、<<、&=、||=、*=、>>=、<<=
    • 专用运算符有
      • 强制运算符：()
      • 下标运算符：[]
      • 成员运算符：->、.
      • 指针运算符；&、*、+、-
  • 算术运算符对数据进行简单的算术运算，字符型数据可以看成整形数据，可以参加算数运算；
    • 除运算／时如果两个运算对象都是整形，商是整数，有小数则截去小数部分，如果有一个运算对象是实型，商是实型；
    • 模运算%的对象必须是整形数据，结果是整除后的余数，运算结果的符号必须与被除数相同；
  • 加1、减1运算，前缀是先加／减1后使用对象，后缀是先使用对象再加／减1；
  • 关系运算符用于比较两个数据大小，逻辑值真、假使用整数1、0表示，字符数据比较按字符对应的ASCII码值的大小比较；
  • 逻辑运算符对关系式、逻辑值、整形、实型、字符型等数据进行运算，运算结果是逻辑值；
  • 赋值运算符分为（基本）赋值运算符、算术自反赋值运算符、位自反赋值运算：
  • 位运算符分为：位逻辑运算符、位移位运算符、位自反赋值运算符：
    • 位逻辑运算符：
      • 位非（取反）：～，对二进制数每一位执行逻辑反操作，1反为0，0反为1；
      • 位与：&，两个等长二进制数同位都为1则结果值为1；
      • 位或：|，两个等长二进制数同位一位为1则结果值为1；
      • 按位加（位异或）：^，两个等长二进制数同位不同值结果值为1，同则结果值为0；
    • 位移位运算符，移位时，移出的位数全部丢弃，移出的空位补入的数与左移或右移有关；
      • 左移则补入数全部为0；
      • 右移又与是否带符号有关：
        • 带符号则补入的数全部等于原数的最左边位上的数；
        • 不带符号则补入的数全部为0；

表达式

  • 表达式的构成规则：
    1. 单个的常量、变量、函数调用；
    2. 单目前缀运算符表达式；
    3. 表达式 单目后缀运算符；
    4. 表达式 双目运算符表达式；
    5. 表达式？表达式：表达式；
    6. 有限次使用上述规则获得的运算式；
  • 表达式分为：算术表达式、关系表达式、逻辑表达式、条件表达式、赋值表达式和逗号表达式；

数据类型转换规则

  • 表达式计算中数据类型的自动转换规则：参加运算的各个数据都转换成数据长度最长的数据类型，计算出的结果值的类型就是数据长度最长的数据类型；
  • 运算结果存入变量时数据类型的自动转换规则：先将运算结果的数据类型自动转换成左边变量的数据类型，然后再赋予该变量；
  • 运算结果的强制性数据类型转行规则：在需要进行类型转换的表达式前面加上圆括号括住的数据类型符，如果表达式时单个常量或变量则不需要用圆括号括起，如果是则需要；

数组

二维数组的初始化

  • 给所有元素赋值：int a[2][2] = {{1, 3}, {2, 4}}；
  • 不分行给所有元素赋值：int a[2][2] = {1, 2, 3, 4}；
  • 只对每行前若干元素赋值：int a[4][3] = {{1}, {2}}；
  • 只对前若干行的前若干元素赋值：int a[3][2] = {{1, 2}, {3, 4}}；
  • 若给所有行的前若干元素赋值，行长度可以省略：int a[][2] = {1, 2, 3, 4, 5, 6} 或 int a[][2] = {{1, 2}, {3, 4}, {5, 6}}，行长度都是3；

字符数组和字符串

  • 字符串的最后一个字符是字符串结束标记符'\0'；
  • 使用scanf("%s", 字符数组)输入字符串会把回车换行符和空格符当作输入结束，结束后的输入不会存入字符数组；
  • 字符串处理的系统函数

指针

  • 指针是一种数据类型，用于存放数据的内存单元地址；
  • 当一个变量值占用一个字节时，该字节的地址就是变量的地址；当变量占用连续的若干字节时，第一个字节的地址（该字节的地址值是所- 占用的连续字节中地址值最小的）就是该变量的地址；
  • 通过变量名查取变量的地址，再从变量对应地址的字节中取得值或将值存入变量对应地址的字节中称为直接存取方式；
  • 一个数组占用的总字节数等于数组长度 X 每个数组元素占用的字节数；
  • 数组的首地址就是第一个元素的地址；
  • 数组元素a[i]的地址 = 数组首地址 + i * 数组元素的数据类型所占用的字节数；
  • 指针型变量的数据类型是它将要指向的变量或数组的数据类型；
  • 地址型表达式：运算结构是地址型数据的表达式；
  • 一维数组名可以看成是地址常量，其值是一维数组的首地址；
  • “*指针变量名“代表它指向的变量或数组元素；
  • 取地址运算符& 的运算对象必须是已定义过的变量或数组元素，不可以是数组名；
  • 指针运算符* 的运算对象必须是地址，不能是整数或非地址型常量；

指向一维数组的指针变量

  • 指针变量指向一维数组后，对下标为i的数组元素引用一共有下列四种方法：*(指针变量 + i) *(数组名 + i) 指针变量[i] 数组名[i] ，前两种称为指针法，后两种称为下标法；
  • 对指向一维数组的指针进行算术运算时，所加减的整数不是指绝对的地址整数值，而是该指针变量向后或向前运动的同类型一维数组元素的数目；
  • 指针变量的减法运算要求两个必须指向同一个数组，结果为指向的数组元素的下标的差；
  • 关系运算符可以用于比较地址值的大小，比较对象可以是指针变量、地址常量；

指向字符串的指针变量

  • 允许直接使用字符串常量，可以用字符型指针变量指向它，以后可以使用字符型指针变量来处理这个字符串或字符串的单个字符；

函数

函数概念

  • 建立函数称为“函数的定义“；使用函数称为“函数的调用“；每一个函数都能完成特定的任务，称为“函数的功能“；每次调用函数时的某些待处理的数据和函数运行后获得的某些结果统称为“函数的参数“；带入函数的对象称为“函数参数的输入“；函数运行的结果带回调用函数称为“函数参数的输出“；函数参数的输入和输出统称“函数间数据的传递“；
  • 函数的分类：
    • 从定义上分为用户函数和系统函数；
    • 从形式上分为有参函数和无参函数；
    • 从返回值上分为有返回值函数和无返回值函数；
    • 从函数的作用范围上分为外部函数和内部函数；
  • 函数分为函数头 和函数体：
    • 函数头分为：存储类型符、数据类型符、函数名、形式参数表；
      • 存储类型符有extern和static，可以省略，省略时默认为extern；
        • extern定义的函数称为外部函数，可以被其他编译单位的中的函数调用；
        • static定义的函数称为内部函数，只能被本编译单位中的函数调用；
      • 函数名是一个标识符，在同一个编译单位中函数不能重名；
    • 函数体分为：数据定义语句序列、执行语句序列；
  • 函数间传递数据有：值传递方式、地址传递方式、返回值方式、全局变量传递方式；
    • 值传递方式传递的是参数值，特点是“参数值的单向传递“；形參是指针变量属于值传递，函数调用时需要为指针变量分配内存，结束时回收指针变量占用的内存，但是函数执行期间对指针变量的值所做的操作会保留；
    • 地址传递方式传递的是地址，实參和形參都是地址型。调用函数时，将实际参数的地址赋予对应的形式参数数组作为其首地址，即使用相同的内存。
    • 返回值传递方式要求函数头中要有数据类型符并在函数体的最后返回值；
    • 全局变量传递方式使用在函数外定义的外部变量；

变量的存储类型

  • 变量的存储类型分为：auto自动型、register寄存器型、static静态型、extern外部参照型：
    • auto自动型、堆栈型，在省略存储类型符的情况下默认为自动型，分配在内存的堆栈区。
      • 当一个变量被声明就会在堆栈区分配内存用于存放变量，函数调用结束后变量占用的内存会被释放供下一个变量使用；
    • register寄存器型，分配在CPU的通用寄存器中；
      • 寄存器型变量有数量限制，超出设为自动变量；
      • 占用字节数多的变量不能定义为寄存器型变量；
      • 寄存器型变量的作用域在函数内
  • static静态型，分配在内存的数据区中；
    • 在程序开始时已经分配好内存，程序结束时释放内存；
    • 不进行初始化则赋予“零值”，数值型赋予0，字符型赋予'\0'；
  • extern外部参照型，用于多个编译单位之间传递数据；
    • 不允许初始化；

变量的生存期和作用域

  • 变量被分配内存至被收回内存期间称为“生存期”，变量可以使用的程序区域成为变量的作用域；；
  • 在函数外声明的变量称为“外部变量”，在函数内声明的变量称为“内部变量”；
  • 生存期从定义到程序结束的变量称为“全局变量“，生存期只覆盖某个函数的称为”局部变量“；
  • 外部变量只能定义成无存储类型或static，内部变量可以定义成auto、register、static；
  • static的内部变量的作用域是全局，尽管在离开定义的函数不能被使用但依然存在，直到程序结束才会被释放内存；

函数的嵌套调用和递归调用

  • 嵌套调用是函数A调用函数B，函数B再调用函数C；递归调用是函数调用自己本身；
  • 向下的递归调用过程称为“递归过程”，向上的携带返回值计算返回表达式的过程称为“回溯过程”；

指针型函数

  • 指针型函数指函数的返回值是指针型，接受返回值的必须是指针变量、指针数组元素等；
  • 指针型函数的函数名前需加上*；

  3. 定义无名称结构型的同时定义变量；

成员的引用

  • 结构型变量的地址和变量的成员的地址都可以使用；
  • 成员的引用，例：结构型变量名.成员名；
  • 结构型变量成员地址的引用方法：
    • 成员是变量形式的变量地址：&结构型变量名.成员名；
    • 成员是数组形式的数组首地址：结构型变量名.成员名；
    • 成员是数组，其数组元素的地址：&结构型变量，成员数组[下标]；
  • 结构型变量地址的引用方法：&结构型变量名；

指向结构型数据的指针变量的定义和使用

  • 调用方法：(*指针变量).成员名 指针变量->成员名；
  • "*指针变量"必须用圆括号括住，因为“*”的优先级低于“.”，如果不加是语法错误：*指针变量.成员名；
  • 定义结构型是，其成员的变量可以是本结构型但必须是指针变量或指针数组；
  • 当结构型数组的首地址已经赋予相同结构结构型的指针变量，调用方法：(*(指针变量+i)).成员名 (指针变量+i)->成员名；

用户自定义类型

  • typedef 类型符 类型符;，第一个类型符可以是基本数据类型符或用户自定义的无名称结构性；第二个类型符是用户自选的标识符，通常建议全大写；
  • 指针型自定义：typedef 类型符 * 类型符;，例：typedef float * TF，与 float * f 等价；

文件

  • 如果文件存放的是数据称为“数据文件”，如果存放的是源程序清单或是编译连接后生成的可执行程序称为“程序文件”；
  • 从磁盘文件读取数据到内存：文件的打开；内存中的数据存入磁盘文件：文件的关闭；
  • 内存区是磁盘文件和程序中存放数据的变量、数组之间交换数据的缓冲区域，称为：文件缓冲区；
  • 计算机的输入/输入出设备的作用是输入/输出数据，其功能和文件的读取数据/写入数据相似，所以把输入/输出设备称为设备文件；
  • 键盘称标准输入设备；显示器称标准输出设备和标准错误输出设备；
  • 从输入设备上读取数据，可以看成是从输入设备文件中读数据；将数据写到输出设备上，可以看成是写到输出设备文件中；
  • 文件类型是一种特殊的“结构型”，其成员记录了处理文件是所需的信息：
    • 文件代号int _fd；
    • 文件缓冲区剩余的字节数int _cleft；
    • 文件操作模式int _mode；
    • 下一个待处理字节的地址char *nextc；
    • 文件缓冲区首地址char *buff；
  • 用“FILE”定义的指针变量称为“文件型指针”；
